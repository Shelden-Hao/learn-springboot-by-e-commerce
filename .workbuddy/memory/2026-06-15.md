# 2026-06-15

## Feature 1: 商品查询 REST API
- 添加 `spring-boot-starter-web` 依赖
- 创建 `Product.java` 模型类（BigDecimal 金额、LocalDateTime）
- 创建 `ProductController.java`（@RestController + @GetMapping）
- 返回硬编码商品列表 JSON

## Feature 2: JPA + H2 数据库
- 添加 `spring-boot-starter-data-jpa` + `h2` 依赖
- Product 改造为 @Entity（JPA 注解）
- 创建 `ProductRepository`（extends JpaRepository）
- 创建 `DataInitializer`（CommandLineRunner 种子数据）
- ProductController 注入 Repository，新增 GET /{id} 单商品查询
- application.yaml 配置 H2 + JPA（show-sql + format_sql + H2 Console）

## Feature 3: 新增商品 + 请求体校验
- 添加 `spring-boot-starter-validation` 依赖
- 创建 `dto/CreateProductRequest`（DTO 分层，含 @NotBlank/@NotNull/@DecimalMin 等校验注解）
- ProductController 新增 `POST /api/products`（@Valid + @RequestBody + @ResponseStatus 201）
- 教学重点：Entity vs DTO 分离、校验注解机制、@Valid 拦截流程

## Feature 4: 更新 + 删除 → CRUD 完整体
- 创建 `dto/UpdateProductRequest`（所有字段可选，部分更新语义）
- Product.java 新增 `applyUpdate()` 方法（只覆盖非 null 字段）
- ProductController 新增 PUT /{id} 和 DELETE /{id}
- 引入 `@Transactional` 事务注解（查询→修改→保存原子化）
- @ResponseStatus：POST→201 / DELETE→204 / PUT→200

## Feature 5: 统一异常处理 + 分页查询
- 创建 `exception/ProductNotFoundException`（语义化自定义异常，替代通用 RuntimeException）
- 创建 `exception/GlobalExceptionHandler`（@RestControllerAdvice + @ExceptionHandler 精确路由）
  - ProductNotFoundException → 404
  - MethodArgumentNotValidException → 400 + 字段级错误列表
  - Exception（兜底）→ 500
- 创建 `dto/ErrorResponse`（统一错误 JSON 结构：status/message/timestamp/errors）
- ProductController 分页改造：`findAll()` → `findAll(Pageable)`，返回 `Page<Product>`
- DataInitializer 种子数据扩展到 15 条（分页效果更直观）
- 所有 RuntimeException → 替换为 ProductNotFoundException
